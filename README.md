Distributed Cache System Using LRU & Consistent Hashing
This project aims to develop a distributed caching system similar to those used in large-scale
web applications. It uses hash maps for fast key-value access, LRU (Least Recently Used)
cache eviction policy for memory management, and consistent hashing to distribute data across
multiple cache nodes.
Learning Outcomes:
● Deep understanding of hashing techniques
● Implementation of LRU cache using HashMap + Doubly Linked List
● Understanding consistent hashing algorithm
● Handling collisions and load balancing
● Knowledge of distributed systems concepts
● Time and space complexity optimization\
What it takes to execute this project:
● Implement LRU cache mechanism
● Design hash table for storing key-value pairs
● Implement consistent hashing ring
● Simulate multiple cache servers
● Handle cache miss and eviction policies
● Add performance benchmarking
● Implement concurrency handling (optional advanced feature)
● Provide CLI interface for testing
Real-world application:
● Web application caching (like Redis/Memcached concepts)
● Content delivery networks
● Cloud-based distributed systems
● Database query caching
